# `scripts_of_material`

**Type**: Array of strings

---

The `scripts_of_material` field describes the scripts used in the record. We model this by creating a **`rico:Language`** entity for each script listed in the AtoM array. These entities are then linked to the record using the **`rico:hasOrHadLanguage`** property.

This model connects the following pieces of information:

  - **The record resource**: The record.
  - **The script**: A `rico:Language` entity representing a specific script.
  - **The script name**: The literal script name from AtoM is attached to the `rico:Language` entity using `rico:note` with a "Script: " prefix to maintain semantic clarity.

### RiC-O Entities

1.  `rico:Language` (the script(s) used in the record)

### Example

```turtle
# Define the Language entity for Latin script
:Script_Latin a rico:Language ;
  rico:note "Script: Latin"@en .

# Define the Language entity for Arabic script
:Script_Arabic a rico:Language ;
  rico:note "Script: Arabic"@en .

# Example record set entity
:RecordSet_123 a rico:RecordSet ;
  rico:hasOrHadLanguage :Script_Latin ;
  rico:hasOrHadLanguage :Script_Arabic .
```
