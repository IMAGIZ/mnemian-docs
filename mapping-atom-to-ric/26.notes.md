# `notes`

**Type**: String

---

The `notes` field in AtoM (ISAD 3.6.1 General note) contains an array of unstructured note strings about the resource (record). We model this using simple literal values on the record resource via the **`rico:note`** property, one per retrieved (AtoM) note. 

This adheres to the "simplest available model" principle, preserves original text, and avoids over-modeling. If, in future, specific types of notes need differentiation, we can introduce additional properties or typed notes in a separate enhancement.

### RiC-O Entities

1.  `rico:RecordSet/Record/RecordPart` (the record resource)
