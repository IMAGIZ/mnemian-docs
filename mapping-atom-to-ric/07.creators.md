# `creators`

**Type**: Array of objects

---

The `creators` field in AtoM describes the agents responsible for creating a record. We model this using a **`rico:CreationRelation`** entity, which is an n-ary relation that formally represents the relationship between a record and its creator. This relation uses the proper RiC-O n-ary structure to link a record to an agent, which is either a **`rico:Person`**, **`rico:CorporateBody`**, **`rico:Family`**, or **`rico:Mechanism`**.

## Enhanced Actor Processing

Our implementation now uses a **two-phase approach** to process creators:

1. **Phase 1**: Process all other informationobject metadata first
2. **Phase 2**: Process creators last, using detailed actor data from the AtoM actor API

This approach ensures that:
- Creator entities are **deduplicated** across multiple records
- **Rich actor metadata** is captured from the dedicated actor API endpoint
- **Consistent entity relationships** are maintained

### Actor Deduplication Strategy

Each creator is identified by a unique `id` field from the AtoM informationobject API. Before creating a new actor entity, we:

1. Check if an actor with that `id` already exists in the database
2. If exists: reuse the existing actor entity
3. If not exists: fetch detailed actor data and create full entity structure

### Actor API Integration

When a new actor is encountered, we fetch detailed information from:
```
GET /api/actors/{slug}
```

This provides rich metadata including:
- Multiple name variants (authorized, parallel, standardized, other names)
- Reference codes and identifiers
- Dates of existence
- Legal status and general context
- Detailed biographical/historical information

## RiC-O Entity Model

### Core Entities

1. **`rico:CreationRelation`** - The n-ary relationship between record and creator
2. **`rico:Person`**, **`rico:CorporateBody`**, **`rico:Family`**, or **`rico:Mechanism`** - The creator agent
3. **`rico:AgentName`** - Multiple name entities for each actor
4. **`rico:Identifier`** - Reference codes and other identifiers
5. **`rico:Date`** - Dates of existence
6. **`rico:LegalStatus`** - Legal status information
7. **`rico:Instantiation`** - Digital instantiation metadata

### Name Variants

Each actor can have multiple name entities linked via `rico:hasOrHadAgentName`:

- **Authorized form of name** (ISAAR 5.1.2) - Primary name
- **Parallel names** (ISAAR 5.1.3) - Alternative forms in different languages
- **Standardized names** (ISAAR 5.1.4) - Normalized forms
- **Other names** (ISAAR 5.1.5) - Additional variants

Each name entity includes:
- `rico:textualValue` - The actual name text
- `rdfs:label` - Descriptive label indicating the name type

### Example

```turtle
# Example record set entity
:RecordSet_123 a rico:RecordSet .

# Define the creator (Person) entity with rich metadata
:Person_JohnSmith a rico:Person ;
  rico:history "This is a long text multiline biographical history of John Smith."@en ;
  rico:note "General context: Additional biographical context information."@en ;
  rico:hasOrHadAgentName :Name_JohnSmith_Authorized ;
  rico:hasOrHadAgentName :Name_JohnSmith_Parallel ;
  rico:hasOrHadIdentifier :Identifier_JohnSmith ;
  rico:isOrWasActiveAtDate :Date_JohnSmith ;
  rico:hasOrHadLegalStatus :LegalStatus_JohnSmith ;
  rico:hasOrHadDigitalInstantiation :Instantiation_JohnSmith .

# Authorized form of name
:Name_JohnSmith_Authorized a rico:AgentName ;
  rico:textualValue "Smith, John"@en ;
  rdfs:label "Authorized form of name (ISAAR 5.1.2)"@en .

# Parallel name
:Name_JohnSmith_Parallel a rico:AgentName ;
  rico:textualValue "Smith, J."@en ;
  rdfs:label "Parallel name (ISAAR 5.1.3)"@en .

# Reference code identifier
:Identifier_JohnSmith a rico:Identifier ;
  rico:textualValue "SMITH-J"@en ;
  rico:hasIdentifierType :IdentifierType_ReferenceCode .

:IdentifierType_ReferenceCode a rico:IdentifierType ;
  rico:textualValue "Reference code"@en .

# Dates of existence
:Date_JohnSmith a rico:Date ;
  rico:expressedDate "1920 to 1995"@en ;
  rico:normalizedValue "1920/1995"@en .

# Legal status
:LegalStatus_JohnSmith a rico:LegalStatus ;
  rico:note "Public domain"@en .

# Digital instantiation
:Instantiation_JohnSmith a rico:Instantiation ;
  rico:carrierType "Digital"@en ;
  rico:structure "Structured metadata, held in Access to Memory (AtoM) database"@en ;
  dc:source "https://atom.example.com/api/actors/smith-john-1920-1995"@en .

# Define the creation relation using n-ary structure
:CreationRelation_123 a rico:CreationRelation ;
  relationHasSource :Person_JohnSmith ;
  relationHasTarget :RecordSet_123 .
```

### Note
- The properties used in the above example follow the RiC-O n-ary relation structure. The `rico:relationHasSource` and `rico:relationHasTarget` properties connect the relation to the source (creator) and target (record) entities.*
