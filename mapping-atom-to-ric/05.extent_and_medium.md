# `extent_and_medium`

**Type**: String

---

The **`rico:Extent`** entity is used to represent the quantity of a record or record set, usually expressed in physical or digital terms. This maps directly to the `extent_and_medium` field in AtoM. We model this by creating a `rico:Extent` entity, which holds the literal value from AtoM, and then linking it from the `rico:RecordSet` or `rico:RecordPart` using `rico:hasExtent`.

The `rico:Extent` entity connects the following pieces of information:

  - **The resource it describes**: The `rico:RecordSet` or `rico:RecordPart` to which the extent and medium applies, which links out using `rico:hasExtent`.
  - **The textual value**: In the **`rico:textualValue`** datatype property, attached directly to the `rico:Extent` entity.
  - **The type of extent**: A relationship to a **`rico:ExtentType`** entity, which can categorize the extent (e.g., "linear metres," "digital files," "items"). This is useful if the AtoM field contains structured information that can be parsed.

### RiC-O Entities

1.  `rico:Extent` (represents the quantity and medium)
2.  `rico:ExtentType` (shared, if a type can be inferred from the value)

### Example

```turtle
# Define the ExtentType (shared) entity and its label
:ExtentType_Boxes a rico:ExtentType ;
  rdfs:label "Boxes"@en .

# Create the extent entity
:Extent_456 a rico:Extent ;
  rico:textualValue "3 boxes"@en ;
  rico:hasExtentType :ExtentType_Boxes .

# Example record set entity (links to the extent)
:RecordSet_123 a rico:RecordSet ;
  rico:hasExtent :Extent_456 .
```
