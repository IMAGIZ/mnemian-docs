# `scope_and_content`

**Type**: String

---

The `scope_and_content` field in AtoM contains a summary of the record's content, which aligns with the definition of `rico:scopeAndContent`. This property is a direct datatype property on a record resource, making it the most straightforward and semantically accurate way to model this information. `rico:scopeAndContent` provides a concise way to capture a narrative description of the materials, adhering to our "simplest available model" constraint.

The `rico:scopeAndContent` property is defined as:

  * **Domain**: `rico:RecordResource` (which includes `rico:RecordSet`, `rico:Record`, `rico:RecordPart`). This ensures it can be used on all levels of description.
  * **Range**: `rdfs:Literal`. This means it holds a free-text string, which is perfect for a narrative field like scope and content.
