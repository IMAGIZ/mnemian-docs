# `dates`

**Type**: Array of objects

---

The `dates` field in AtoM contains structured temporal information about a record. We model this using the **`rico:Date`** entity, which represents a specific time period. The AtoM data provides a human-readable `date` string as well as `start_date` and `end_date` values, which we will use to populate the `rico:expressedDate` and `rico:normalizedValue` properties on the `rico:Date` entity.

We only (currently) model date types 'Creation' or 'Accumulation', and we use the direct properties **`rico:hasCreationDate`** and **`rico:hasAccumulationDate`** to link the core record to the `rico:Date` entity. This is the simplest and most compliant model for the given data, as it directly connects the record to its specific type of date without an intermediary `rico:DateType` entity.

### RiC-O Entities

1.  `rico:Date` (the specific time period)

### Example

```turtle
# Example record set entity
:RecordSet_123 a rico:RecordSet .

# Create the date entity - note EDTF format for date ranges
:Date_XYZ a rico:Date ;
    rico:expressedDate "January 1, 2015 - December 31, 2016"@en ;
    rico:normalizedValue "2015-01-01/2016-12-31" .

# Link the record set to the date using the appropriate property
:RecordSet_123 rico:hasCreationDate :Date_XYZ .
```
