# `institution_identifier`

**Type**: String

---

The `institution_identifier` field in AtoM contains a unique identifier for the repository institution that holds the record. We model this as a **`rico:Identifier`** entity attached to the existing repository `rico:CorporateBody` entity (created via the `repository` field processing). This approach leverages the existing repository entity while adding the institutional identifier information.

The modeling pattern:
- **The repository entity**: The existing `rico:CorporateBody` entity representing the repository (created from the `repository` field).
- **The institutional identifier**: A `rico:Identifier` entity containing the textual value via `rico:textualValue`.
- **The identifier type**: A shared `rico:IdentifierType` entity labeled "Repository identifier"@en.
- **The relationship**: The `rico:CorporateBody` links to the identifier using `rico:hasOrHadIdentifier`.

This approach reuses the existing repository entity while adding the institutional identifier as additional metadata, keeping the model simple and avoiding duplication.

### RiC-O Entities
1. `rico:CorporateBody` (existing repository entity; links out via `rico:hasOrHadIdentifier`)
2. `rico:Identifier` (institutional identifier, textual value from AtoM)
3. `rico:IdentifierType` (shared, reused; labeled "Repository identifier")

### Example

```turtle
# Define the IdentifierType (shared) entity and its label
:IdentifierType_Repository  a  rico:IdentifierType ;
    rdfs:label "Repository identifier"@en .

# Existing repository entity (created from repository field)
:CorporateBody_ICA a rico:CorporateBody ;
    rico:hasOrHadName :Name_ICA ;
    rico:hasOrHadIdentifier :RepositoryIdentifier_ICA .

# Repository identifier entity
:RepositoryIdentifier_ICA  a  rico:Identifier ;
    rico:textualValue "ICA-001"@en ;
    rico:hasIdentifierType  :IdentifierType_Repository .
```
