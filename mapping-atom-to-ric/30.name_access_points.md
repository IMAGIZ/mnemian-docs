# `name_access_points`

**Type**: Array of strings

---

The `name_access_points` array in AtoM contains zero-or-more person or corporate body names as plain strings, used to aid discovery. In RiC-O v1.1, we model these as **`rico:Person`** or **`rico:CorporateBody`** entities with associated **`rico:Name`** entities containing the textual values. This approach provides proper semantic modeling of named entities while maintaining simplicity.

The modeling pattern:
- Create one `rico:Person` or `rico:CorporateBody` entity per unique name (using the same logic as creators)
- Create one `rico:Name` entity per agent, containing the textual value via `rico:textualValue`
- Link each record resource to the relevant agent(s) using **`rico:hasOrHadSubject`**
- Reuse existing agents if there's a 100% name match to avoid duplication

This approach leverages RiC-O's native agent modeling capabilities while keeping the implementation simple and avoiding duplication of agent entities.

Notes:
- `rico:hasOrHadSubject` domain is `rico:RecordResource`; range is `rico:Thing`. `rico:Person` and `rico:CorporateBody` can safely be used as objects.
- `rico:hasOrHadName` connects an agent to its `rico:Name` entities.
- `rico:textualValue` is the datatype property for storing the textual value in `rico:Name` (since `rico:Name` ⊂ `rico:Appellation`).

### RiC-O Entities
1. `rico:RecordSet/Record/RecordPart` (described resource; links out via `rico:hasOrHadSubject`)
2. `rico:Person` or `rico:CorporateBody` (one per distinct agent)
3. `rico:Name` (one per agent, containing the textual value)

### Example

```turtle
# Two agent entities created from the AtoM array
:Person_SmithJohn a rico:Person ;
  rico:hasOrHadName :Name_SmithJohn .

:Person_ClareSmith a rico:Person ;
  rico:hasOrHadName :Name_ClareSmith .

# Name entities containing the textual values
:Name_SmithJohn a rico:Name ;
  rico:textualValue "Smith, John"@en .

:Name_ClareSmith a rico:Name ;
  rico:textualValue "Clare Smith"@en .

# Link a record resource to the agents as subjects
:RecordSet_123 a rico:RecordSet ;
  rico:hasOrHadSubject :Person_SmithJohn , :Person_ClareSmith .
```

### Notes on name access point modeling
- This approach creates shared `rico:Person` or `rico:CorporateBody` entities that can be reused across multiple records, avoiding duplication.
- Each agent has exactly one `rico:Name` containing the textual value from AtoM.
- Uses the same name-to-agent-type logic as the creators field for consistency.
- Future development may add biographical information, agent types, or hierarchical agent relationships without changing the basic record-to-agent links.
