# `place_access_points`

**Type**: Array of strings

---

The `place_access_points` array in AtoM contains zero-or-more place names as plain strings, used to aid discovery. In RiC-O v1.1, we model these as **`rico:Place`** entities with associated **`rico:PlaceName`** entities containing the textual values. This approach provides proper semantic modeling of geographic locations while maintaining simplicity.

The modeling pattern:
- Create one `rico:Place` entity per unique place name
- Create one `rico:PlaceName` entity per place, containing the textual value via `rico:textualValue`
- Link each record resource to the relevant place(s) using **`rico:isAssociatedWithPlace`**

This approach leverages RiC-O's native place modeling capabilities while keeping the implementation simple and avoiding duplication of place entities.

Notes:
- `rico:isAssociatedWithPlace` domain is `rico:Thing`; range is `rico:Place`. `rico:RecordResource` can safely use this property.
- `rico:hasOrHadPlaceName` connects a `rico:Place` to its `rico:PlaceName` entities.
- `rico:textualValue` is the datatype property for storing the textual value in `rico:PlaceName` (since `rico:PlaceName` ⊂ `rico:Name` ⊂ `rico:Appellation`).

### RiC-O Entities
1. `rico:RecordSet/Record/RecordPart` (described resource; links out via `rico:isAssociatedWithPlace`)
2. `rico:Place` (one per distinct place)
3. `rico:PlaceName` (one per place, containing the textual value)

### Example

```turtle
# Two place entities created from the AtoM array
:Place_1 a rico:Place ;
  rico:hasOrHadPlaceName :PlaceName_1 .

:Place_2 a rico:Place ;
  rico:hasOrHadPlaceName :PlaceName_2 .

# PlaceName entities containing the textual values
:PlaceName_1 a rico:PlaceName ;
  rico:textualValue "Place 1"@en .

:PlaceName_2 a rico:PlaceName ;
  rico:textualValue "New York, USA"@en .

# Link a record resource to the places
:RecordSet_123 a rico:RecordSet ;
  rico:isAssociatedWithPlace :Place_1 , :Place_2 .
```

### Notes on place modeling
- This approach creates shared `rico:Place` entities that can be reused across multiple records, avoiding duplication.
- Each place has exactly one `rico:PlaceName` containing the textual value from AtoM.
- Future development may add geographic coordinates, place types, or hierarchical place relationships without changing the basic record-to-place links.
