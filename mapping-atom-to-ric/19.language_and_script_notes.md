# `language_and_script_notes`

**Type**: Array of strings

---

The `language_and_script_notes` field describes additional information about the languages and scripts used in the record. We model this by creating a **`rico:Language`** entity to hold the narrative text. This entity is then linked to the record using the **`rico:hasOrHadLanguage`** property.

This model connects the following pieces of information:

  - **The record resource**: The record.
  - **The language notes**: A `rico:Language` entity representing the language and script information.
  - **The narrative text**: The literal text from AtoM is attached to the `rico:Language` entity using `rico:generalDescription` to provide detailed information about language and script usage.

### RiC-O Entities

1.  `rico:Language` (the language and script notes for the record)

### Example

```turtle
# Define the Language entity for language and script notes
:LanguageNotes_123 a rico:Language ;
  rico:generalDescription "The records contain both English and French text, with Latin script used for official documents and Arabic script for personal correspondence. Some documents include handwritten annotations in German."@en .

# Example record set entity
:RecordSet_123 a rico:RecordSet ;
  rico:hasOrHadLanguage :LanguageNotes_123 .
```
