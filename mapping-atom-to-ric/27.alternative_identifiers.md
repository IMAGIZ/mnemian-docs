# `alternative_identifiers`

**Type**: (array)

---
The `alternative_identifiers` array in AtoM provides zero-or-more additional identifiers for the same resource, each typically including a human label and an identifier string. In RiC-O v1.1, each alternative identifier is modeled as a distinct `rico:Identifier` and linked from the resource using `rico:hasOrHadIdentifier`, with its value in `rico:textualValue`. All such identifiers are typed with a shared `rico:IdentifierType` whose `rdfs:label` is "Alternative"@en; we create only one of these and reuse it across all records.

When AtoM provides a per-identifier descriptive label, we preserve it as a simple note on the `rico:Identifier` (using `rico:note`). This keeps the model simple while retaining the narrative label without introducing custom subtypes.

### RiC-O Entities
1. `rico:RecordSet/Record/RecordPart` (identified resource; links out via `rico:hasOrHadIdentifier`)
2. `rico:Identifier` (one per alternative identifier)
3. `rico:IdentifierType` (shared, reused; labeled "Alternative")

### Example

```turtle
# Define the IdentifierType (shared and reused)
:IdentifierType_Alternative  a  rico:IdentifierType ;
    rdfs:label "Alternative"@en .

# Example record set entity (identified resource)
:RecordSet_123  a  rico:RecordSet ;
    rico:hasOrHadIdentifier  :AltIdentifier_1 , :AltIdentifier_2 .

# Create multiple alternative identifiers and link them to the record via rico:hasOrHadIdentifier
:AltIdentifier_1  a  rico:Identifier ;
    rico:textualValue "ALT-ID1"@en ;
    rico:note "Alternate identifier label 1"@en ;
    rico:hasIdentifierType  :IdentifierType_Alternative .

:AltIdentifier_2  a  rico:Identifier ;
    rico:textualValue "ALT-ID2"@en ;
    rico:note "Alternative identifer label 2"@en ;
    rico:hasIdentifierType  :IdentifierType_Alternative .
```
