# `languages_of_material`

**Type**: Array of strings

---

The `languages_of_material` field describes the languages used in the record. We model this by creating a **`rico:Language`** entity for each language listed in the AtoM array. These entities are then linked to the record using the **`rico:hasOrHadLanguage`** property.

This model connects the following pieces of information:

  - **The record resource**: The record.
  - **The language**: A `rico:Language` entity representing a specific language.
  - **The ISO code**: If a corresponding ISO 639-1 code can be found, we attach it to the `rico:Language` entity using `rico:name` as the canonical identifier.
  - **The language name**: The literal language name from AtoM is attached to the `rico:Language` entity using `rdfs:label` for human-readable display.

### RiC-O Entities

1.  `rico:Language` (the language(s) used in the record)

### Example

```turtle
# Define the Language entity for English
:Language_en a rico:Language ;
  rico:name "en" ;
  rdfs:label "English"@en .

# Define the Language entity for French
:Language_fr a rico:Language ;
  rico:name "fr" ;
  rdfs:label "French"@en .

# Example record set entity
:RecordSet_123 a rico:RecordSet ;
  rico:hasOrHadLanguage :Language_en ;
  rico:hasOrHadLanguage :Language_fr .
```
